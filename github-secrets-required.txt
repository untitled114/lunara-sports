# GitHub Secrets Required for Lunara App Deployment

## Required GitHub Repository Secrets

### Azure Authentication
AZURE_CREDENTIALS
  Description: Service principal for Azure access
  Format: JSON object with clientId, clientSecret, subscriptionId, tenantId
  Example: {
    "clientId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "clientSecret": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "subscriptionId": "c581bf76-aaa0-414b-9b77-828fe1bcdba4",
    "tenantId": "9b87166a-59e0-4f83-a904-fca5873efdfa"
  }

AZURE_STATIC_WEB_APPS_API_TOKEN
  Description: For Static Web Apps deployment
  Location: Azure Portal > Static Web Apps > Lunara > Manage deployment token
  Format: String token

### Django Application Settings
DJANGO_SECRET_KEY
  Description: Production Django secret key
  Format: Long random string (50+ characters)
  Generate with: python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"

## How to Add Secrets to GitHub Repository

1. Go to: https://github.com/untitled114/lunaro.io/settings/secrets/actions
2. Click "New repository secret"
3. Add each secret name and value from above
4. Save each secret

## Service Principal Creation (for AZURE_CREDENTIALS)

Run these Azure CLI commands:

```bash
# Create service principal
az ad sp create-for-rbac --name "lunara-app-deploy" \
  --role contributor \
  --scopes /subscriptions/c581bf76-aaa0-414b-9b77-828fe1bcdba4 \
  --sdk-auth

# This will output the JSON for AZURE_CREDENTIALS secret
```

## Static Web Apps Token

1. Go to Azure Portal
2. Navigate to Static Web Apps > Lunara
3. Go to "Manage deployment token"
4. Copy the token value for AZURE_STATIC_WEB_APPS_API_TOKEN

## Notes

- All secrets are case-sensitive
- Database credentials are from your existing Neon setup
- Generate a new DJANGO_SECRET_KEY for production (different from dev)
- Keep these values secure and never commit them to the repository
